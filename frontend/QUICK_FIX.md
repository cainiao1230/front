# 🎯 快速检查清单

## 现在试试这些步骤：

### 第一步：清除旧缓存
```bash
# 在浏览器控制台执行：
localStorage.clear()
location.reload()
```

### 第二步：重新登录
- 用户名：`admin`
- 密码：`admin123`

### 第三步：验证菜单
检查左侧菜单是否出现：
- [ ] ✅ 系统设置 （文件夹图标）
  - [ ] 用户管理
  - [ ] **角色权限** ← 您要找的页面！
  - [ ] 操作日志

### 第四步：如果还是看不到
访问诊断工具：
```
http://localhost:5173/home/debug
```

在诊断工具中查看：
- 📦 localStorage 数据（确认 `userInfo` 存在）
- 🔐 角色权限检查（确认角色是 `admin`）
- ✅ 权限验证结果（确认 system 权限是 ✅）

---

## 🔧 如果诊断工具显示权限不足：

### 原因排查：

1️⃣ **角色不是 admin**
   - ❌ 重新确认登录使用的是 `admin` 用户
   - ✅ 检查诊断工具中的"角色"显示是否为 `admin`

2️⃣ **localStorage 中的 userInfo 为空**
   - ❌ 登录时数据未正确保存
   - ✅ 在诊断工具点击"清除本地缓存"后重新登录

3️⃣ **role 字段值大小写不对**
   - ❌ 可能返回了 `Admin` 或 `ADMIN`
   - ✅ 已修复，现在会自动转换为小写

---

## 📱 直接访问角色权限页面：

如果上述都不行，可以直接在浏览器地址栏访问：
```
http://localhost:5173/home/system/roles
```

---

## 💡 最重要的一点：

**重新登录一遍！** 键名修复后，必须重新登录才能生效。

旧的 localStorage 数据是基于错误的键名存储的，不会自动更新。
